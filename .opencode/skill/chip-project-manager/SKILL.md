# 芯片项目经理 (Chip Project Manager)

## 概述

本技能为芯片/半导体项目的管理提供专门化的指导和工具，帮助项目经理高效管理芯片设计、验证、量产等全流程。

## 使用场景

当用户请求以下任务时，应使用此技能：
- 芯片项目的需求分析和规格定义
- 设计评审和技术决策支持
- 项目进度跟踪和里程碑管理
- 风险评估和缓解措施
- 供应链和制造协调
- 成本管理和预算控制
- 团队协作和沟通管理
- 质量管理和失效分析

## 芯片项目生命周期管理

### 1. 项目启动阶段
- 需求收集和规格定义
- 可行性分析和技术评估
- 项目团队组建
- 初步项目计划制定

### 2. 设计阶段
- 架构设计评审
- RTL设计审查
- 综合/布局/验证规划
- IP核选型和集成

### 3. 验证阶段
- 功能验证计划
- FPGA原型验证
- 仿真和形式验证
- 时序分析和收敛
- DFT/DFM检查

### 4. 量产准备
- T0/T1试产
- 良率分析
- 封装和测试规划
- 量产释放标准

### 5. 量产和维护
- 质量监控
- 客户支持
- 缺陷追踪和修复
- 版本迭代

## 关键工作流程

### 需求规格书编写
1. 收集客户/市场需求
2. 定义性能指标（功耗、面积、成本）
3. 制定验证计划
4. 确定IP选型策略
5. 编写技术规格文档

### 设计评审组织
1. 准备设计文档和检查清单
2. 组织跨部门评审会议
3. 记录评审意见和行动项
4. 跟踪问题解决状态
5. 评审总结和决策记录

### 风险管理
1. 识别技术风险（工艺偏差、时序违例等）
2. 识别供应链风险（产能、交期）
3. 评估影响和优先级
4. 制定缓解措施
5. 定期复盘和更新风险矩阵

### 项目跟踪
- 使用甘特图跟踪关键路径
- 每周状态报告
- 里程碑达成情况
- 资源利用率分析
- 预算vs实际支出

## 行业标准和最佳实践

### 设计阶段
- 遵循设计方法论（自顶向下/自底向上）
- 完整的设计文档
- 版本控制和变更管理
- 设计规则检查（DRC/LVS）

### 验证阶段
- UVM验证方法学
- 回归测试覆盖
- 形式验证
- 跨时钟域检查

### 量产阶段
- 良率提升（Yield Enhancement）
- ATE测试优化
- 失效分析流程
- 客户现场支持

## 注意事项

- 芯片项目周期长（通常18-24个月），需要长期规划
- 工艺节点选择需考虑成本和性能权衡
- IP核采购是关键风险点
- 供应链波动需要提前预警
- 跨职能团队沟通至关重要
