# Chip Frontend Design

## 概述

此技能为芯片前端IP设计工程师提供专门化指导，涵盖从模块规格到RTL代码交付的完整流程。前端设计是芯片开发的关键环节，确保模块正确集成到SoC并满足时序、功耗和面积约束。设计工程师负责产出高质量RTL代码和完整的交付物文档，供下游验证团队进行验证。

## 使用场景

当用户请求以下任务时，应使用此技能：
- 芯片模块接口定义和规格
- 协议接口实现（AXI、AHB、PCIe、DDR等）
- 时序分析和约束编写（SDC）
- RTL代码编写和代码审查
- 综合和时序收敛
- 标准单元设计和集成
- 低功耗设计技术
- 可测试性设计（DFT）
- 规格文档拆解和Feature List输出

## 前端设计流程

### 1. 规格理解和接口定义

- 阅读模块规格文档
- 理解功能需求和性能指标
- 定义模块边界和接口信号
- 与架构师对齐接口协议
- 确认时钟域和复位策略

### 2. 协议实现

- 总线接口实现（AXI/AHB/CHI）
- 中断控制器设计
- DMA接口设计
- 调试接口（JTAG/APB）
- 电源管理接口

### 3. RTL代码编写

- 遵循编码规范和风格指南
- 参数化设计支持复用
- 时钟域清晰划分
- 异步处理（CDC）
- 异常和错误处理逻辑

### 4. 时序约束编写

- 时钟定义和生成
- 输入延迟约束
- 多周期路径约束
- 虚假路径约束
- 时钟组约束

### 5. 综合

## 关键设计原则

### 接口设计

### AXI接口

**关键信号：**
- **通道**: Write Address (AW), Write Data (WD), Read Address (AR), Read Data (RD), Write Response (WR), Read Response (RR)
- **事务类型**: Read, Write, Atomic, Exclusive, Locked
- **突发传输**: 支持可配置突发长度
- **QoS**: 服务质量和优先级支持

**设计考虑：**
- 握手协议（READY/VALID）
- 流水线传输
- 突发边界处理
- 超时和重试机制

### 时钟域设计

**CDC（Clock Domain Crossing）处理：**
- **同步器设计**: 2触发器或3触发器安全同步
- **FIFO深度**: 足够处理最坏情况延迟
- **握手协议**: 握手信号完整（req/ack）
- **异步复位**: 确保所有域正确复位

**格雷码转换：**
- 使用格雷码减少跨时钟域的亚稳态风险
- 适合多位计数器指针

### 时序设计

**时序路径类型：**
1. **Register-to-Register**: 路径短，时序容易满足
2. **Input-to-Register**: 取决于输入延迟约束
3. **Register-to-Output**: 需要输出负载建模
4. **Clock-gated Path**: 时钟门控延迟计入

**时序优化：**
- 流水线插入（关键路径）
- 寄存器复制（扇出控制）
- 逻辑重组（平衡路径）
- 复位树优化

## 标准单元设计

### 标准单元类型

**触发器**：
- D触发器
- JK触发器
- T触发器
- 电平触发器（Latch）

**组合逻辑**：
- AND/OR/NAND/NOR
- XOR/XNOR
- 多路复用器（MUX）
- 编码器/解码器

**存储单元**：
- D触发器阵列
- 存储器元件（SRAM位）

### 标准单元设计规则

**设计规范：**
- **库使用**: 必须使用工艺库的标准单元
- **驱动强度**: 足够驱动最大扇出负载
- **时钟树**: 平衡skew和负载
- **输入约束**: 考虑最大输入转换时间
- **输出负载**: 考虑最大输出负载

**低功耗设计：**
- **门控时钟**: 未使用的触发器门控
- **数据门控**: 未使用的逻辑门控
- **多阈值电压**: 支持不同工作模式

### 存储器设计

**寄存器文件考虑：**
- **访问时间**: 单周期vs多周期
- **写策略**: Write-through vs Write-back
- **复位策略**: 同步vs异步
- **低功耗**: 时钟门控和睡眠模式

**RAM接口：**
- **地址/数据总线**: 宽度和时序
- **控制信号**: CE, WE, OE等
- **读/写周期**: 时序关系
- **字节使能**: 支持部分字写入

**设计考虑：**

### 外设接口

## 综合和时序收敛

### Design Compiler流程

**综合步骤：**
1. **读取设计**: RTL和约束文件
2. **映射**: RTL到标准单元
3. **优化**: 面积和时序优化
4. **链接**: 生成网表
5. **报告**: 生成面积、时序、功耗报告

**时序收敛策略：**
1. **初步时序分析**: 识别关键路径
2. **约束调整**: 调整时钟周期和路径约束
3. **逻辑修改**: 优化关键路径
4. **迭代综合**: 直到时序满足
5. **布局后时序**: 与后端团队协作

### DRC/LVS检查

**DRC（Design Rule Check）:**
- 最小/最大宽度违规
- 最小间距违规
- 天线效应
- 电源网络完整性
- 信号电平冲突

**LVS（Layout vs Schematic）:**
- 电路连接正确性
- 器件类型匹配
- 参数一致性
- 电源/地网络连接

## 规格文档拆解与Feature List输出

### Feature List概述

**定义：** Feature List（功能特性列表）是将架构规格文档拆解为可验证的功能点清单，是连接前端设计与后端验证的关键交付物。每个Feature应具备独立性、可验证性和完整性。

**输出目标：** 输出给下游前端验证工程师(DV)，作为UVM验证环境开发和测试用例设计的依据。

### 规格文档拆解流程

1. **原文提取**
   - 通读架构Spec文档，标注所有功能描述
   - 识别性能指标、接口协议、操作模式
   - 标记边界条件和异常场景

2. **特性归类**
   - 按功能模块分类（控制逻辑、数据通路、接口等）
   - 按优先级排序（Must-have / Should-have / Could-have）
   - 识别跨模块依赖关系

3. **Feature定义**
   - 每个Feature对应一个独立验证点
   - 明确输入激励、预期行为、验收标准
   - 标注依赖的时钟、复位、配置条件

### Feature List文档结构

```markdown
# Feature List - [模块名称]

## 基本信息
- **模块名称**:
- **版本**:
- **作者**:
- **创建日期**:
- **关联Spec版本**:

## Feature列表

| ID | Feature描述 | 优先级 | 验证类型 | 依赖项 | 验收标准 |
|----|------------|--------|----------|--------|----------|
| FE_001 | [功能描述] | P1 | 功能/异常 | 无 | [PASS条件] |

### FE_001: [Feature名称]
- **描述**: [详细说明]
- **前置条件**: [配置/状态要求]
- **测试步骤**:
  1. [激励序列]
  2. [观察响应]
- **预期结果**: [正确行为]
- **边界条件**: [极端场景]
- **关联需求**: [对应Spec条款]
```

### Feature分类标准

**按验证类型分类：**
- **功能Feature**: 正常操作模式下的功能验证
- **异常Feature**: 错误注入、边界条件、超时处理
- **配置Feature**: 寄存器配置、模式切换
- **性能Feature**: 吞吐量、延迟、带宽指标
- **时序Feature**: 建立/保持时间、时钟门控

**按优先级分类：**
- **P0 (Critical)**: 核心功能，芯片必备
- **P1 (High)**: 重要特性，功能完整必需
- **P2 (Medium)**: 增强功能，可接受延后
- **P3 (Low)**: 锦上添花，资源允许时实现

### Feature List交付标准

**完整性检查：**
- 覆盖Spec中所有功能性描述
- 覆盖所有接口协议特性
- 覆盖所有操作模式
- 覆盖边界和异常场景

**可验证性检查：**
- 每个Feature有明确的PASS/FAIL标准
- 激励可生成，响应可观测
- 无歧义描述，避免模糊用语

**交付物清单：**
- Feature List文档（Markdown/Excel）
- 与Spec的追溯矩阵
- 优先级和进度评估
- 预估验证资源（测试用例数、仿真时间）

## 代码质量和审查

### 编码规范

**命名约定：**
- 信号名: `signal_name_n`（低位到高位）
- 时钟: `clk_`或`clk_domain_`
- 复位: `rst_n_`或`reset_`
- 常量: `UPPER_CASE_WITH_UNDERSCORE`

**注释要求：**
- 模块头部注释（功能、输入、输出）
- 关键逻辑解释
- 时序相关注释
- 参数说明

### 代码审查清单

| 检查项 | 说明 | 状态 |
|----------|------|------|
| 时钟域清晰 | 无隐式时钟交叉 |  |
| 复位完整 | 所有寄存器可复位 |  |
| 异常处理 | 所有错误场景覆盖 |  |
| 时序约束 | 关键路径有约束 |  |
| 可综合 | 无综合禁止结构 |  |
| 可测试性 | 测试点充分 |  |
